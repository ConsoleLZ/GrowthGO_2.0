## 第一次使用配置用户名和邮箱
#### 配置用户名
```sh
git config --global user.name "yourname"
```

#### 配置邮箱
```sh
git config --global user.email "youremail@xxx.com"
```

#### 查看当前的配置信息
```sh
git config --global --list
```



## 远程仓库

#### 列出所有远程仓库的简写名称

```sh
git remote
```

#### 显示远程仓库的详细 URL 地址

```sh
git remote -v
```

#### 添加一个远程仓库

```sh
git remote add <shortname> <url>
```

#### 重命名

```sh
git remote rename <old-name> <new-name>
```

#### 删除

```sh
git remote rm <remote-name>
```



## 高频命令

#### 查看日志

```sh
git log
```



#### 克隆

```sh
# 克隆远端仓库到本地
git clone <git url>

# 克隆远端仓库到本地，并同时切换到指定分支 branch1
git clone <git url> -b branch1

# 克隆远端仓库到本地并指定本地仓库的文件夹名称为 my-project
git clone <git url> my-project
```



#### 推送流程

```sh
# 将所有修改的文件都提交到暂存区
git add .

# 将工作区内容提交到本地仓库，并添加提交信息 your commit message
git commit -m "your commit message"

# 将当前本地分支 branch1 内容推送到远程分支 origin/branch1
git push <远程仓库名称> <远程仓库分支>
```



#### 拉取

```sh
git pull <远程仓库名称> <远程仓库分支>
```



## 分支操作

#### 切换到已有的本地分支

```sh
git checkout branch1
```

#### 分支创建并切换

```sh
git checkout -b branch2
```



## 回退代码

#### 临时切换到某次提交

```sh
git checkout <commit-hash>
```

#### 回退到某个提交（放弃该提交后的提交记录和修改）

```sh
git reset --hard <commit-hash>
```
